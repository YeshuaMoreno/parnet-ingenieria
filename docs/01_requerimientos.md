# 1. Requerimientos del Sistema

## 1.1 Descripción General
El sistema ParNet Ingeniería es una aplicación web desarrollada para la gestión de productos, servicios, solicitudes y visualización de contenido corporativo.

## 1.2 Requerimientos Funcionales

- RF1: El sistema debe permitir visualizar información corporativa (Inicio, Servicios, Productos, etc.)
- RF2: El sistema debe permitir consultar productos desde la base de datos
- RF3: El sistema debe permitir generar reportes en PDF y Excel
- RF4: El sistema debe permitir registrar solicitudes de servicio
- RF5: El sistema debe permitir mostrar noticias dinámicas
- RF6: El sistema debe permitir navegación sin recarga (SPA)
- RF7: El sistema debe permitir acceso a panel administrativo
- RF8: El sistema debe permitir CRUD de productos
- RF9: El sistema debe registrar visitas al sitio

## 1.3 Requerimientos No Funcionales

- RNF1: Interfaz basada en diseño web legacy (tipo 2008)
- RNF2: Tiempo de respuesta menor a 2 segundos
- RNF3: Compatibilidad con navegadores modernos
- RNF4: Uso de arquitectura cliente-servidor
- RNF5: Uso de API REST
